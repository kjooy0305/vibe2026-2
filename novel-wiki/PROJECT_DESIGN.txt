【 이 문서 사용법 】
  - Claude는 모든 작업 시작 전 이 파일을 읽어 전체 구조를 파악한 뒤 수정 시작.
  - 작업 완료 후 변경 사항을 이 파일에 반영.
  - 미완성 기능 또는 사용자가 추가로 요청했으나 미반영된 기능을 파악하여 알려줄 것.

================================================================================
1. 앱 개요
================================================================================

  목적: 웹소설(게임 판타지 탑 등반물) 세계관 창작 관리 앱
  기술: 완전 오프라인 SPA (Vanilla JS + IndexedDB + Service Worker)
  배포: GitHub Pages → https://kjooy0305.github.io/vibe2026-2/novel-wiki/
  저장소: github.com/kjooy0305/vibe2026-2 (master 브랜치 → GitHub Pages)

  커밋/배포 원칙:
    - 항상 master:master 로 push (git push origin master:master)
    - sw.js CACHE_VERSION 을 반드시 버전업 후 커밋 (현재: novel-wiki-v88)
    - 배포 결과 링크: https://kjooy0305.github.io/vibe2026-2/novel-wiki/

================================================================================
2. 스크립트 로드 순서 (index.html) — 의존성 있음
================================================================================

  로드 순서 (반드시 이 순서 유지):
    1. js/db.js          — IndexedDB (window.DB)
    2. js/store.js       — AppStore (window.AppStore, QUEST_POOL, MISSION_POOL)
    3. js/router.js      — AppRouter (window.AppRouter)
    4. js/utils.js       — Utils, AppFlags (window.Utils, window.AppFlags)
    5. js/constants.js   — AppConstants (window.AppConstants)
    6. js/search.js      — SearchEngine (window.SearchEngine)
    7. pages/*.js        — 각 페이지 모듈 (의존성 아래 참조)
    8. js/app.js         — 부트스트랩 (페이지 등록 + 앱 시작)

  pages/ 내 의존성:
    - home.js 반드시 shortcuts.js 보다 먼저 로드 (HOME_ALL_PAGES 전역 상수)
    - tower.js 반드시 tower-floor-form.js, tower-subfloor-form.js 보다 먼저 로드
      (Object.assign(window.Pages.tower, {...}) 패턴으로 확장)
    - organizations.js 반드시 organizations-form.js 보다 먼저 로드
    - characters.js 반드시 characters-form.js 보다 먼저 로드
    - stat-defs.js 는 어느 시점에든 로드 가능
      (window.StatDefs 전역, characters-form/gates-form/tower-floor-form 에서 참조)

  js/db.js
    역할: IndexedDB 래퍼. window.DB 전역.
    DB_NAME: 'NovelWikiDB', DB_VERSION: 12

    STORES (28개):
      worlds, characters, skills, achievements, organizations, constellations,
      gates, monsters, towers, items, jobs, events, worldRules, templates,
      folders, settings, streak, countries, companies, keywordFolders,
      keywords, statDefs, traps, places, gods, races, quests, editHistory

    worldId 인덱스 있는 스토어 (22개):
      characters, skills, achievements, organizations, constellations, gates,
      monsters, towers, items, jobs, events, worldRules, folders, countries,
      companies, keywordFolders, keywords, statDefs, traps, places, gods,
      races, quests

    worldId 인덱스 없는 스토어:
      worlds, templates, settings, streak, editHistory

    HISTORY_STORES (23개, DB.put 시 자동 스냅샷):
      worlds, characters, skills, items, monsters, organizations, gates,
      towers, countries, companies, places, races, gods, quests, constellations,
      events, worldRules, traps, jobs, statDefs, keywords, achievements,
      keywordFolders

    DB.put() 히스토리 로직:
      - 기존 레코드 있을 때만 스냅샷 (신규 생성 시 히스토리 없음)
      - getSetting('historyMaxLength', 366) 으로 최대 보관 수 읽음
      - 초과 시 오래된 것부터 삭제 (비동기, fire-and-forget)
      ※ DB.put()은 항상 data.id, data.updatedAt, data.createdAt 자동 설정

    API: open, genId, getAll(store, worldId?), get(store, id), put(store, data),
         del(store, id), getSetting(key, default), setSetting(key, value),
         exportAll, importAll

  js/store.js
    역할: 전역 상태 관리 + 스트릭/미션 엔진. window.AppStore 전역.

    내부 상태 (AppStore._state):
      currentWorldId, worlds, currentWorld, streak, searchScope

    QUEST_POOL (14개 일일 퀘스트):
      각 퀘스트: { id, store, isNew, texts[], cleared[], hint }
      id 목록: add_character, edit_character, add_skill, edit_skill,
               add_achievement, add_item, edit_item, add_monster,
               add_gate, edit_gate, write_novel, add_event, add_quest, add_job
      날짜 기반 선택: seed = date.getFullYear()*10000+... % QUEST_POOL.length

    MISSION_POOL (14개 8시간 미션):
      각 미션: { id, text, target, store, isNew?, reward }
      8시간마다 새 3개 미션 세트 (getMissionState())

    SHIELD_COST: 50포인트

    recordActivity(storeName, isNew):
      - 오늘의 퀘스트가 storeName+isNew 와 일치하면 퀘스트 클리어
      - 현재 미션 중 storeName+isNew 와 일치하는 것 progress 증가
      - _onQuestDone(): streak.count++, streak.history 갱신, +50점, totalCleared++
      - _onMissionDone(): 미션 완료 시 streak 1증가 (단, missionAdvancedStreak 중복 방지)
      ※ 호출하는 페이지: characters-form, skills-form, achievements, items-form,
                        monsters-form, constellations-form, countries-form, countries(inline),
                        companies(2곳), event-graph, places-form, jobs-form, keywords,
                        quests(3곳), traps
      ※ 호출 안 하는 페이지 (미션 기여 없음):
                        gates-form, organizations-form, tower-floor-form,
                        tower-subfloor-form, stat-defs, world, world-rules,
                        reminders, timeline, family-tree
      ※ updateStreak만 호출 (recordActivity 없음): races, gods, novel-view

    updateStreak():
      - home.js init, novel-view.js 저장 시, races.js 저장 시, gods.js 저장 시 호출
      - 날짜 변경 감지 → 쉴드 소비 또는 스트릭 리셋

    getMissionState(): 현재 8시간 미션 세트 반환 (만료 시 새로 생성)
    getMissionResetCost(state): 미완료 미션당 15포인트
    buyShield(): 50포인트 소비 → 쉴드 +1
    resetMissions(): 포인트 소비 → 미션 새로고침

  js/router.js
    역할: 해시 기반 SPA 라우터. window.AppRouter 전역.

    PAGE_TITLES 맵 (32개):
      home, world, characters, organizations, skills, achievements, constellations,

  js/utils.js
    역할: 유틸리티 함수 모음. window.Utils, window.AppFlags 전역.

    Utils 주요 API:
      toast(msg, type, duration): 3초 토스트 (success/error/info/warning)
      confirm(title, desc, onOk, okLabel): #confirmDialog 기반 확인 다이얼로그
      confirmWithInput(title, desc, expected, onOk): 텍스트 입력 확인 (삭제 시)
      openModal(title, body, onConfirm, confirmLabel): #globalModalOverlay 기반 모달
        - onConfirm에서 false 반환 시 모달 유지 (검증 실패 시)
        - onConfirm에서 true 반환 시 모달 닫힘
      closeModal(): #globalModalOverlay.classList.remove('open')
        ※ 모달 닫을 때 반드시 closeModal() 사용
        ※ #globalModal (모달 박스 자체)의 open 클래스 제거는 동작 안함
      fieldError(id): 입력 필드 빨간 테두리 표시
      escHtml(str): XSS 방지 HTML 이스케이프
      nl2br(str): \n → <br>
      gradeColor(grade): 등급별 CSS 색상
      gradeBadge(grade): 등급 배지 HTML
      formatDate(ts): 타임스탬프 → '2026-05-17 14:30' 형식
      matchesQuery(text, query): 검색어 매칭 (공백 분리 AND)
      imageToBase64(file): 이미지 파일 → base64 문자열
      addFieldToggles(root): textarea 앞 form-label 있는 모든 필드에 접기/펼치기 버튼 자동 추가

    AppFlags:
      - localStorage의 'appFlags' JSON에 동기 저장/읽기
      - AppFlags.get(key, default), AppFlags.set(key, value), AppFlags.getAll()
      - 페이지 초기화 시 DB.getSetting('flag_'+key) 와 동기화 필요
      - 빠른 동기 접근용 (폼 렌더링 시)

    역할: 전역 상수 관리. window.AppConstants 전역.

    DEFAULTS (코드 내장 기본값):
      grades: ['F','E','D','C','B','A','S','SS','SSS','EX']

    const_* 설정 키: grades, attributes, skillSeries, itemTypes, iconPresets,
                     statCategories (+ 기타 커스텀 상수)

    역할: 전체 검색 엔진. window.SearchEngine 전역.

    renderResults(results, query): 결과 HTML 생성

5. 주요 패턴 & 규칙
================================================================================

  페이지 등록: window.Pages.pageName = { init(container, opts), destroy() }
  파일 분리:   Object.assign(window.Pages.pageName, { methods }) — 분리된 form 파일

  모달:
    Utils.openModal(title, bodyHTML, onConfirm, label)
    → onConfirm에서 false 반환 시 모달 유지 (필드 검증 실패)
    → onConfirm에서 true 반환 시 자동으로 Utils.closeModal() 호출
    → Utils.fieldError(id) 로 빨간 테두리 표시
    ⚠️ 모달 수동 닫기 시 반드시 Utils.closeModal() 사용 (closeModal은 #globalModalOverlay 조작)
       #globalModal.classList.remove('open') 사용 금지 (동작 안 함)

  토글 칩:
    data-active="true|false" 방식 (wave-cond-chip, concept-chip 등)
    체크박스 사용 금지 (시각 피드백 없어 혼란 유발)

  접기/펼치기:
    section-toggle 버튼 + section-body div
    _collapsedSections 객체로 상태 추적

  등급 색상: Utils.gradeColor(grade) / Utils.gradeBadge(grade)
  이미지:    Utils.imageToBase64(file) → base64 저장 (IndexedDB)

  히스토리:
    DB.put() 내부 자동 스냅샷 (HISTORY_STORES 대상, 기존 레코드 있을 때만)
    editHistory: { id, storeName, itemId, itemName, timestamp, snapshot }
    최대 보관 수: historyMaxLength 설정 (기본 366)

  세계 선택:
    AppStore.getCurrentWorldId() → wid (null이면 세계 선택 화면 표시)

  기능 플래그:
    AppFlags.get(key, default) — 동기 접근, localStorage 캐시
    DB.getSetting('flag_'+key) — 비동기 DB 접근 (settings.js에서 저장)
    둘 다 항상 동시 업데이트 필요 (AppFlags.set + DB.setSetting)

  스트릭/활동 기록:
    저장 성공 후: await AppStore.updateStreak(); await AppStore.recordActivity(storeName, isNew);
    updateStreak() — 날짜 변경 감지 (항상 호출)
    recordActivity() — 미션/퀘스트 진행 (저장 페이지에서 호출)
    ⚠️ 일부 페이지 미호출 현황은 섹션 3 참조

  검색 자동완성 드롭다운:
    blur 이벤트 + setTimeout 200ms 패턴으로 결과창 닫기
    mousedown에서 e.preventDefault() 로 blur 보다 먼저 처리
