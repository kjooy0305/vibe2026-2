  - 작업 완료 후 변경 사항을 이 파일에 반영.
  - 미완성 기능 또는 사용자가 추가로 요청했으나 미반영된 기능을 파악하여 알려줄 것.

================================================================================
1. 앱 개요
================================================================================

  목적: 웹소설(게임 판타지 탑 등반물) 세계관 창작 관리 앱
  기술: 완전 오프라인 SPA (Vanilla JS + IndexedDB + Service Worker)
  배포: GitHub Pages → https://kjooy0305.github.io/vibe2026-2/novel-wiki/
  저장소: github.com/kjooy0305/vibe2026-2 (master 브랜치 → GitHub Pages)

  커밋/배포 원칙:
    - 항상 master:master 로 push (git push origin master:master)
    - sw.js CACHE_VERSION 을 반드시 버전업 후 커밋 (현재: novel-wiki-v88)
    - 배포 결과 링크: https://kjooy0305.github.io/vibe2026-2/novel-wiki/

2. 파일 구조 및 역할
================================================================================

  [ 루트 파일 ]
    index.html      — 앱 셸 HTML. 스크립트 로드 순서 중요 (의존성 있음)
    manifest.json   — PWA 매니페스트
    sw.js           — 서비스 워커 (캐시 버전: CACHE_VERSION, 현재 v88)

  [ js/ — 핵심 모듈 ]
    js/db.js        — IndexedDB 래퍼 (DB.open, DB.get, DB.getAll, DB.put, DB.del,
                      DB.getSetting, DB.setSetting, DB.genId)
                      DB_VERSION=12, editHistory 스토어 포함
                      DB.put() 에서 주요 스토어 변경 시 자동 히스토리 스냅샷 저장
    js/store.js     — AppStore (세계 선택, 스트릭, 활동 기록, 기능 플래그)
    js/router.js    — AppRouter (해시 기반 SPA 라우터, 뒤로가기 스택)
    js/app.js       — 앱 부트스트랩, 서랍 메뉴, 헤더, 페이지 등록, 12초 로딩 타임아웃
    js/utils.js     — Utils (toast, confirm, openModal, escHtml, gradeColor 등)
                      AppFlags: 동기 기능 플래그 캐시 (localStorage 기반)
                        AppFlags.get(key, default) / AppFlags.set(key, value)
    js/constants.js — AppConstants (기본값, 등급 목록 등)
                      AppConstants.load() → this._C 로 상수 비동기 로드
    js/search.js    — SearchEngine (전체 검색, 타입 필터 칩 UI: 전체/캐릭터/스킬/업적 등)
                      search(query, worldId, scopeAll, typeFilter)

  [ css/ ]
    css/main.css        — 기본 변수, 레이아웃, 타이포그래피
    css/components.css  — 버튼, 카드, 배지, 모달, 서랍 등 컴포넌트
    css/mobile.css      — 하단 내비게이션, 반응형
    css/novel-view.css  — 소설 뷰어 전용

  [ pages/ — 페이지 모듈 (window.Pages.xxx) ]

    ── 홈 & 탐색 ──
    home.js           — 홈 화면 (스트릭, 미션, 바로가기 북마크 5슬롯, 전체페이지 그리드,
                        편집 이력 아이콘버튼 → 이전 버전 모달)
    shortcuts.js      — 바로가기 전용 페이지 (하단탭 마지막 칸, 북마크+전체페이지 그리드)
    timeline.js       — 타임라인 (사건 추가/편집 가능, 카드 클릭 → 상세 모달 → 편집 버튼)
    event-graph.js    — 사건 그래프
                        · 캔버스 기반 노드-엣지 그래프 (드래그/핀치 줌 지원)
                        · 노드색상: BASE_COLORS + 커스텀 컬러 피커 (+저장 버튼)
                        · 연결(outcomes): 화살표 방향만, 타입 없음, label+edgeColor만 저장
                        · 기존 데이터 호환: EDGE_COLORS 로 구버전 out.type 색상 fallback
                        · 회귀 시스템 플래그(useRegression) on/off 시 회차 필드·필터·노드텍스트 연동
                        · 사건 필터: 회차 칩 + 기준사건 깊이 필터
    family-tree.js    — 가족 관계도
    world-rules.js    — 세계관 규칙

    ── 세계 & 시스템 ──
    world.js          — 세계/차원 관리
    settings.js       — 설정 (아이콘 관리, 기능 플래그, 데이터 내보내기/가져오기,

3. IndexedDB 스키마 (DB_VERSION=12)
================================================================================

  스토어 (모든 스토어는 keyPath:'id'):
    worlds, characters, skills, achievements, constellations, gates, towers,
    monsters, timeline (=events), organizations, countries, companies,
    places, traps, items, jobs, reminders, keywords, statDefs, races, gods, quests
    editHistory (히스토리 스냅샷, worldId 인덱스 없음)

  설정 (DB.getSetting / DB.setSetting — key-value):
    homeBookmarks           — 홈 북마크 5슬롯 배열
    historyMaxLength        — 편집 히스토리 최대 저장 수 (기본 366, settings.js에서 저장)
    gateCustomTypes_{wid}   — 게이트 커스텀 종류 목록
    gateCustomBreakTypes_{wid} — 게이트 커스텀 브레이크 유형
    flag_{key}              — 기능 플래그 (gradeColors, showAuthorBadge, useRegression 등)
    iconList_{cat}          — 카테고리별 아이콘 목록 (world/const/org/country/company/trap)
    novelQuestions          — 테마 질문 목록
    worldTypes              — 세계 타입 목록
    const_{key}             — 전역 상수 (grades, attributes, etc.)
    searchScope             — 검색 범위 ('world' | 'all')
    eventColors             — 사건 그래프 커스텀 색상 목록 (JSON 문자열)

4. 주요 패턴 & 규칙
================================================================================

  페이지 등록: window.Pages.pageName = { init(container, opts), destroy() }
  파일 분리:   Object.assign(window.Pages.pageName, { methods }) — 분리된 form 파일

  모달:       Utils.openModal(title, bodyHTML, onConfirm, label)
              → onConfirm 에서 false 반환 시 모달 유지 (필드 검증용)
              → Utils.fieldError(id) 로 빨간 테두리 표시

  토글 칩:    data-active="true|false" 방식 (wave-cond-chip, concept-chip 등)
              체크박스 사용 금지 (시각 피드백 없어 혼란 유발)

  접기/펼치기: section-toggle 버튼 + section-body div
               _collapsedSections 객체로 상태 추적

  등급 색상:  Utils.gradeColor(grade) / Utils.gradeBadge(grade)

  이미지:     Utils.imageToBase64(file) → base64 저장 (IndexedDB)

  히스토리:   DB.put() 내부에서 HISTORY_STORES 대상 자동 스냅샷
              editHistory 스토어 → { id, storeName, itemId, itemName, timestamp, snapshot }

  세계 선택:  AppStore.getCurrentWorldId() → wid (null 이면 세계 선택 화면 표시)

  기능 플래그: AppFlags.get(key, default) — 동기 접근, localStorage 캐시
               DB.getSetting('flag_'+key) — 비동기 DB 접근 (settings.js에서 저장)
               형태 모두 동시 지원. 폼/페이지는 AppFlags.get() 사용 권장.
